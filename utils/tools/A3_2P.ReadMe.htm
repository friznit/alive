Arma3Alpha to P:\  from mikero

This batch file peacefully co-exists with arma2p.

Installation

extract rar package to anywhere that gives you a thrill

Caveats:

  • YOU MUST HAVE a P:\ drive
  • YOU MUST HAVE COPIES OF EXTRACTPBO.EXE (and it's dll). these can be found at dev-heaven
    • Read the documentation that comes with the above packages, or, essentially, place both in same folder as this one
  • You must have pre-installed BI's personal tools at some stage

None of this is earth shattering stuff. The only use of this batch file is because you're already using the above tools anyway.
 

Usage

simply click on the cmd file and go make coffee, feed the cat, or hell, talk to your wife.

Implementation

==== binarised configs et al ===========

*ALL* relevent rapified files (configs,missions,rvmats, bisurfs) are derapified back to humanly readable text.


**********************************
WARNING WARNING WARNING
**********************************

  • Arma3P removes P:\A3 before extractions begin. If you have been silly enough to put your private material inside one of BI's 'official' folders........ 'nuff said.
  • User's  P:\folders are NOT AFFECTED
  • Arma2 is NOT AFFECTED except for:
    • p:\bin, p:\core and a few related xml files
      • At this point of alpha development, the alpha simply contains copies of *some* A2 xmls. This batch file extracts and overwrites them anyway.

***************************************************

This batch file currently extracts to the following folders

  • P:\A3
  • P:\WRP_PROJECTS_A3
  • P:\A3_BIN
  • P:\core
  • P:\LangaugeCore
  • P:\LanguageCore_F
  • P:\LangaugeCore_H
  • P:\ca\LanguageMissions

This batch file modifies

  • P:\Bin

Subsequent upgrades to Arma3 will probably alter this list. It is outside the control of this batch file.

P:\A3_Bin

Like it's Arma2p cousin, this batch file overwrites the existing config.bin supplied with 'personal tools' with the one from A3's dta\bin

This is utterly, irrevocably, essential, because the addons you are attempting to use inherit from this config cpp and not the very old, very stale pe tools bin

no other files in bin are touched.

Instead, this batch file makes an A3_BIN folder for your subsequent use should you wish to.

And , the specific reason for wanting to, is to replace the current shaders of the current buldozer.exe with Arma3.exe

It is left to the user to choose to do so.

P:\WRP_PROJECTS_A3

Due to an astonishing oversight in BinPbo you need a copy of A3 configs in each and every island project you create.

Windows Vista and beyond make this 'problem' relatively simply by using MKLINK. WinXP does not have such a feature. WRP_PROJECTS contains the necessary files (and only the necessary files) you need to copy.

A3pbo.lst

At completion, this batch file produces a pbo listing, for your convenience, of all pbo's encountered in the arma3 application. It is not utilised in any way by the batch.
